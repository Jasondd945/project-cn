# Security Policy

如果你发现 `project-cn` 存在安全问题，请不要先公开提交细节。

## 请报告的范围

- 可能导致误改源目录或误改复制后原始文件的问题
- 路径穿越、任意文件写入、危险输出路径解析
- 敏感信息泄露
- 会让状态文件、报告文件或运行产物泄露本机信息的缺陷

## 报告方式

请优先通过 GitHub 私密安全通道或仓库维护者提供的私下联系方式报告。

如果暂时没有私密通道，可先创建不含漏洞细节的普通 Issue，只说明：

- 这是安全问题
- 影响范围
- 需要私下沟通

## 处理原则

- 收到报告后会尽快确认是否可复现
- 在修复完成前，不建议公开完整利用细节
- 修复后会补充说明影响范围和修复版本
