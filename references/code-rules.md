# 代码中文注释增强规则

对 B 类代码文件，生成的是“CN 注释增强版代码副本”，不是翻译代码本身。

## 硬约束

- 不允许改变代码逻辑。
- 不允许修改变量名、函数名、类名、模块名、导入、配置键、协议字段、路径、命令或语法结构。
- 只允许翻译已有英文注释，或补充中文注释。
- 原文件保留不动，新增同目录 `-CN` 代码文件。

## 必须完成的事

- 把已有英文注释、文档字符串、模块说明翻译成简体中文。
- 对没有注释的关键模块、类、函数、关键流程补充中文说明。
- 注释优先说明用途、输入、输出、边界、关键流程和关键分支。

## 风格要求

- 注释准确、克制、可维护。
- 优先帮助人快速看懂代码，不写空话。
- 可以少量加入帮助理解的类比，但只能点到为止，不能文学化。
- 保持原代码格式、缩进和语法有效。

## 自检清单

- 代码逻辑是否完全未改。
- 原有导入和执行顺序是否未动。
- 注释新增后是否仍能通过基本语法检查。
- 是否没有把配置内容、常量值或接口字段误翻译。
