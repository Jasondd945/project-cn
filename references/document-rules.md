# 文档翻译规则

对 A 类纯文档文件，必须生成同目录 `-CN` 副本，并严格遵守下面的约束：

## 必须做的事

- 完整翻译全部自然语言正文。
- 保留原有标题层级、段落顺序、列表顺序、表格结构、引用块和空行布局。
- Markdown 文档必须保留围栏代码块、语言标记、列表缩进、链接 URL、图片路径。
- 结构化文本只翻译 value，不翻译 key。

## 绝对不能做的事

- 不能总结、润色、扩写、缩写、解释。
- 不能把局部翻译当成完整交付。
- 不能翻译代码块内容、行内代码、命令、路径、URL、邮箱、API 名称、包名、模块名、类名、函数名、变量名、标识符、版本号、协议名、配置键名。
- 不能破坏 frontmatter、Markdown 表格、列表缩进和引用结构。

## 处理建议

- 先看完整文件结构，再按标题块或逻辑块分段处理，避免超长文档一次性改坏结构。
- 每次生成后，都对照原文检查：
  - 标题数量是否一致。
  - 代码块数量和语言标记是否一致。
  - 行内代码、路径、URL 是否原样保留。
  - frontmatter key 是否完全未改。

## 命名规则

- `README.md -> README-CN.md`
- `GUIDE -> GUIDE-CN`
- `docs/intro.rst -> docs/intro-CN.rst`
